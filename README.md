# 店铺库存极速快查 WebApp (PWA)

## 📖 项目背景与初衷

传统的店铺 PC 端管理后台过于庞大沉重，店员在仓库、前台或维修工位使用手机直接访问时，面临着**页面加载慢、UI 不适配、操作繁琐**等痛点。

本项目将后台中最核心的“手机型号与配件库存查询”功能独立抽离，重构为一款面向移动端的高清、轻量化 WebApp。通过接入 PWA（渐进式 Web 应用）技术，店员可以将其一键安装至手机桌面，享受媲美原生 App 的极速查询体验。
---

## 🛠️ 技术栈选型

- **核心框架**：React 19 (完全采用全新的 Action 与 Form 状态管理机制)
- **构建工具**：Vite + TypeScript (TSX)
- **样式表现**：Tailwind CSS (自适应布局，完美支持原生级日/夜模式)
- **后端服务**：Supabase (提供高性能的 PostgreSQL 关系型数据库、实时鉴权与联表 API)
- **PWA 支持**：`vite-plugin-pwa` + Workbox
- **托管平台**：Cloudflare Pages (全球边缘 CDN 加速，强安全性 HTTPS 环境)

---

## ✨ 核心功能与移动端优化黑科技

### 1. 彻底解决“刷新闪烁登录页”隐患

利用全局 `appLoading` 状态拦截机制，在 Supabase 鉴权 Session 状态（`getSession`）明确返回前，由现代流线型骨架动画全屏接管，彻底阻断了由于异步网络请求导致的登录页错误闪烁。

### 2. React 19 现代化表单架构 (Action Base)

全面抛弃了旧版繁琐的 `useState` 控件双向绑定，采用 React 19 推荐的 `<form action={formAction}>` 机制。结合内置的 `useActionState` Hook，将 Loading 状态、错误捕获和原生 `FormData` 收集自动化，大幅减少组件重复渲染，提升手机端打字流畅度。

### 3. 高性能前端防抖与一键清空

针对高频输入检索进行了**原生防抖 (Debounce) 处理**：只有当局长停止打字超过 500ms 时才会向 Supabase 发起网络请求。输入框右侧集成了一键清空（Clear）触觉反馈按钮，极大地节约了数据库请求配额并降低了手机功耗。

### 4. 智能无限滚动列表与高性能联表查询

通过限制视口高度与监听原生 `onScroll` 容器，实现下拉触底无感预加载。配件查询采用**反向 Inner Join 过滤策略**，直接由 API 节点平铺返回干净的数组，免去了前端二次数据清洗和循环 `map` 转换，确保弱网设备不卡顿。

### 5. 跨平台 PWA 全局智能安装

在全局 `App.tsx` 中对手机硬件和运行模式（`display-mode: standalone`）进行智能捕获：

- **Android 端**：自动捕获 `beforeinstallprompt` 事件，在顶层滑出原生安装横幅。
- **iOS 端**：由于苹果系统的封闭性，应用会自动渲染一个精致的、带有毛玻璃高斯模糊背景的**底部引导抽屉**，教导店员如何通过 Safari 浏览器将其“添加到主屏幕”。
- 当用户已经从桌面图标启动应用时，所有安装提示将**终生沉浸式隐藏**。

### 6. 边缘离线数据缓存策略

针对 Supabase 数据接口采用 **`NetworkFirst`（网络优先）** 异步缓存策略，并设置 3 秒强超时判定。在地下仓库或信号极差的环境下，应用会自动降级读取 Service Worker 缓存的本地历史库存数据，并弹出红色警示条提示“正在浏览离线数据”，确保业务永不中断。

## 🗄️ 数据库表结构 (Supabase SQL)

项目依赖多对多（Many-to-Many）关系，关系拓扑如下：

```sql
-- 1. 手机型号表
create table models (
  id bigint generated by default as identity primary key,
  brand_id integer not null,
  name text not null unique,
  code text,
  is_tablet boolean default false,
  release_year integer,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. 配件库存表
create table inventory_components (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  sku text not null unique,
  category_id integer,
  quality text,
  cost_price numeric(10, 2),
  suggest_repair_price numeric(10, 2),
  partner_repair_price numeric(10, 2),
  stock_quantity integer default 0,
  supplier_id integer,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. 中间关联兼容表 (手机型号 <-> 配件)
create table component_compatibility (
  id uuid default gen_random_uuid() primary key,
  model_id bigint references models(id) on delete cascade,
  component_id uuid references inventory_components(id) on delete cascade,
  unique(model_id, component_id)
);

```

## 🚀 本地开发指南

### 1. 克隆项目与依赖安装

```bash
git clone <your-repository-url>
cd mobile-repair-assistant
npm install

```

### 2. 配置环境变量

在项目根目录下创建 `.env` 文件，引入你的 Supabase 凭证：

```env
VITE_SUPABASE_URL=https://your-project-id.supabase.co
VITE_SUPABASE_ANON_KEY=your-anonymous-public-key

```

### 3. 运行本地开发环境

```bash
npm run dev

```

> 💡 应用已开启 `devOptions.enabled = true`，在 Dev 模式下你也可以直接在浏览器的 Application 标签页中调试 Service Worker 和控制 PWA 状态。

## 🌐 线上生产部署 (Cloudflare Pages)

本项目完美适配 Cloudflare Pages 持续集成，配置要点如下：

1. **构建命令**：`npm run build`
2. **输出目录**：`dist`
3. **单页应用防 404 路由刷新错位**：项目 `public/` 目录下已预置 `_redirects` 静态控制文件，边缘节点会自动将路由无缝回退至 `index.html`。
4. **Supabase 域名报备**：在 Cloudflare 完成部署并拿到线上域名（`https://xxx.pages.dev`）后，切记前往 Supabase 的 `Authentication -> URL Configuration` 中将该网址添加进 `Site URL` 或重定向白名单中。
